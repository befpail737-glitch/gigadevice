Complete GD32 Migration Guide

Transitioning from STM32 to GigaDevice Microcontrollers

📅 Updated: April 2026 ⏱️ Reading Time: 15 minutes 🎯 Level: Intermediate

Introduction: This comprehensive guide will help you successfully migrate your embedded projects from STM32 to GD32 microcontrollers. As an authorized GigaDevice distributor, Litong has assisted hundreds of customers through this transition. GD32 offers pin-compatible alternatives to STM32 with competitive pricing and often better availability.

💡 Why Migrate to GD32?

  • Cost Savings: 20-40% lower BOM cost compared to equivalent STM32
  • Better Availability: Shorter lead times and stable supply
  • Performance: Equal or better specifications in most cases
  • Pin Compatibility: Drop-in replacement for most applications

1. Migration Overview

2. Step-by-Step Migration Process

1 Hardware Verification

Before starting software migration, verify hardware compatibility:

  • Confirm pin-to-pin compatibility using datasheets
  • Check power supply requirements (typically identical)
  • Verify clock circuitry (GD32 may support higher frequencies)
  • Review reset circuit requirements

💡 Pro Tip

GD32F4 series can typically run at higher clock speeds than equivalent STM32F4. Check if your application can benefit from increased performance.

2 Development Environment Setup

Prepare your development environment for GD32:

  • Download GD32 Firmware Library from GigaDevice website
  • Install GD32 device support packs for your IDE (Keil, IAR, or STM32CubeIDE)
  • Configure debugger settings (J-Link, ST-Link, or DAP-Link compatible)
  • Update flash programming algorithms
// Example: GD32F4xx standard peripheral library structure // Similar to STM32 Standard Peripheral Library #include "gd32f4xx.h" void gpio_config(void) { /* Enable GPIO clock */ rcu_periph_clock_enable(RCU_GPIOA); /* Configure PA0 as output */ gpio_mode_set(GPIOA, GPIO_MODE_OUTPUT, GPIO_PUPD_NONE, GPIO_PIN_0); gpio_output_options_set(GPIOA, GPIO_OTYPE_PP, GPIO_OSPEED_50MHZ, GPIO_PIN_0); }

3 Code Migration

Key areas requiring code modifications:

A. Header Files and Definitions

// STM32 style (before) #include "stm32f4xx.h" #define LED_PIN GPIO_Pin_13 // GD32 style (after) #include "gd32f4xx.h" #define LED_PIN GPIO_PIN_13

B. GPIO Configuration Differences

Operation STM32 (HAL) GD32 (SPL)
Clock Enable __HAL_RCC_GPIOA_CLK_ENABLE() rcu_periph_clock_enable(RCU_GPIOA)
Set Pin High HAL_GPIO_WritePin(GPIOA, GPIO_PIN_0, GPIO_PIN_SET) gpio_bit_set(GPIOA, GPIO_PIN_0)
Toggle Pin HAL_GPIO_TogglePin(GPIOA, GPIO_PIN_0) gpio_bit_write(GPIOA, GPIO_PIN_0, (bit_status)(1-gpio_output_bit_get(GPIOA, GPIO_PIN_0)))

4 Peripheral-Specific Changes

USART/UART Configuration

// GD32 USART initialization example void usart_config(void) { /* Enable clocks */ rcu_periph_clock_enable(RCU_GPIOA); rcu_periph_clock_enable(RCU_USART0); /* Configure TX (PA9) and RX (PA10) */ gpio_af_set(GPIOA, GPIO_AF_7, GPIO_PIN_9); gpio_af_set(GPIOA, GPIO_AF_7, GPIO_PIN_10); gpio_mode_set(GPIOA, GPIO_MODE_AF, GPIO_PUPD_PULLUP, GPIO_PIN_9); gpio_mode_set(GPIOA, GPIO_MODE_AF, GPIO_PUPD_PULLUP, GPIO_PIN_10); /* Configure USART */ usart_deinit(USART0); usart_baudrate_set(USART0, 115200U); usart_receive_config(USART0, USART_RECEIVE_ENABLE); usart_transmit_config(USART0, USART_TRANSMIT_ENABLE); usart_enable(USART0); }

⚠️ Important Note

GD32 USART peripheral may have different register bit definitions compared to STM32. Always refer to GD32 reference manual for exact register configurations.

5 Testing and Validation

Thorough testing is crucial after migration:

  1. Basic Functionality: Verify GPIO, clocks, and basic peripherals
  2. Communication: Test all communication interfaces (UART, SPI, I2C, CAN)
  3. Timing: Verify timing-critical operations (PWM, ADC sampling)
  4. Power: Measure power consumption under various modes
  5. Stress Test: Run extended operation tests

3. Common Issues and Solutions

3.1 Flash Programming Issues

⚠️ Problem: Cannot program GD32 Flash

Solution: GD32 requires different flash programming algorithms than STM32. Download GD32 flash algorithms from GigaDevice website and install in your IDE.

3.2 Clock Configuration

💡 Clock Tree Differences

GD32 clock tree is similar but register names may differ. Use GD32 Clock Configuration Tool (similar to STM32CubeMX) to generate initialization code.

3.3 Interrupt Handling

5. Series-Specific Migration Notes

5.1 GD32F1xx vs STM32F1xx

Most straightforward migration. Peripheral registers are nearly identical. Main differences in flash programming and RCC (clock) configuration.

5.2 GD32F4xx vs STM32F4xx

GD32F4 often supports higher clock speeds (up to 240MHz vs 180MHz for some STM32F4 variants). More SRAM available in equivalent packages.

5.3 GD32E5xx vs STM32G4xx

Both are high-performance series with advanced analog features. Careful review of ADC and comparator configurations recommended.

6. Best Practices

  1. Start with a Peripheral: Migrate one peripheral at a time, test thoroughly before moving to the next
  2. Use Abstraction: Create abstraction layers to minimize future porting effort
  3. Version Control: Maintain separate branches for STM32 and GD32 versions during transition
  4. Documentation: Document all changes made during migration for future reference
  5. Prototyping: Use GD32 evaluation board before committing to PCB redesign

Frequently Asked Questions

Is GD32 compatible with STM32?

Yes, GD32 microcontrollers are pin-to-pin compatible with equivalent STM32 models. The peripherals are highly compatible, making migration straightforward. Most code can be reused with minimal modifications. Litong provides free migration support and code examples.

How do I migrate from STM32F4 to GD32F4?

Migrating from STM32F4 to GD32F4 involves: 1) Verify pin compatibility (they are pin-to-pin compatible), 2) Update startup files and linker scripts, 3) Modify register bit definitions if using direct register access, 4) Update peripheral initialization code for timing differences, 5) Test thoroughly. Litong provides detailed migration guides and FAE support.

What are the main differences between STM32 and GD32?

Main differences: 1) GD32 typically runs at higher clock speeds (e.g., 168MHz vs 168MHz for F4 series), 2) Some peripheral register bit definitions differ, 3) Flash programming algorithms are different, 4) GD32 often has more SRAM in equivalent packages, 5) Price is typically 20-40% lower. Core architecture and instruction set are identical.

Can I use STM32 HAL library with GD32?

STM32 HAL cannot be used directly with GD32 due to register differences. However, GigaDevice provides GD32 Firmware Library (similar to Standard Peripheral Library) and GD32 HAL. The migration effort is similar to porting between different STM32 families. Litong provides code examples and migration templates.
